How Your Website Works

Your website is made up of three separate services that work together. Understanding this will help you manage it going forward.

The Flow

GitHub
Stores the code
→
Vercel
Hosts & deploys it
→
midwestern.construction
Your live domain
Sanity Studio
Edit blog images & content
→
Webhook (/api/bake)
Updates the site
→
Live Website
Reflects changes

Every time you push a change to GitHub, Vercel automatically rebuilds and deploys your site within ~1 minute. Sanity is your content editor — use it to update blog post images.

Step 1 — Create Your Accounts

You need accounts on three platforms. All are free.

GitHub — github.com

1
Go to github.com and sign up with your email.
2
Choose the Free plan.
3
Share your GitHub username with your developer so they can transfer the repository to you.

Vercel — vercel.com

1
Go to vercel.com and click Sign Up.
2
Choose Continue with GitHub — this links the two accounts automatically.
3
Select the Hobby (Free) plan.

Sanity — sanity.io

1
Go to sanity.io and sign up.
2
Your developer will invite you as an Administrator to the existing project. Check your email for the invite and accept it.
3
Once accepted, your content editor is at: midwestern-roofing.sanity.studio

Step 2 — Accept the GitHub Repository Transfer

Your developer will initiate the transfer. You just need to accept it.

1
You'll receive an email from GitHub with the subject "Repository transfer request".
2
Click the link in the email and click Accept Transfer.
3
The repository midwestern-roofing will now appear in your GitHub account.

Step 3 — Set Up Vercel

Import your GitHub repository into Vercel so it can host your site.

1
Log into vercel.com → click Add New Project.
2
Click Import next to midwestern-roofing.
3
Leave all settings as default → click Deploy. Wait for the green checkmark.
4
Go to Settings → Environment Variables and add the following two variables exactly as shown:
Variable Name Value
RESEND_API_KEYre_MXaWRdRG_FFaUjnBQX7AZhR3PoHD2EPS8
GH_TOKENGenerate a new one — see Step 3a below
⚠ Important — GitHub Token: The original GitHub token was shared during development and must be replaced with a new one before going live. Follow Step 3a below to generate a fresh token.

Step 3a — Generate a GitHub Personal Access Token

1
Log into GitHub → click your profile picture (top right) → Settings.
2
Scroll down to Developer settings → Personal access tokens → Tokens (classic).
3
Click Generate new token (classic).
4
Give it a name (e.g. "Vercel Bake"), set expiration to No expiration, and check the repo scope.
5
Click Generate token → copy the token immediately (you won't see it again).
6
Paste this value as the GH_TOKEN environment variable in Vercel.
5
After adding both env vars, go to Deployments → click the three dots on the latest deployment → Redeploy.

Step 4 — Connect Your Domain

Point midwestern.construction (purchased through Squarespace) to your Vercel site.

In Vercel:

1
Go to your project → Settings → Domains.
2
Type midwestern.construction → click Add.
3
Also add www.midwestern.construction → click Add.
4
Vercel will show DNS records — add them in Squarespace as shown below.

In Squarespace:

1
Log into squarespace.com → click Domains → click midwestern.construction.
2
Click DNS Settings → scroll to Custom Records.
3
Add these two records:
TypeName / HostValue
A@76.76.21.21
CNAMEwwwcname.vercel-dns.com
ℹ️ DNS changes can take anywhere from a few minutes to 24 hours to go live. Once active, Vercel will show a green checkmark next to your domain.

Step 5 — Activate Contact Form Emails

The contact form currently sends test emails. To receive real submissions at support@midwestern.construction, complete the steps below.

Verify your domain on Resend:

1
Log into resend.com (account already created during setup) → go to Domains → click Add Domain.
2
Type midwestern.construction → click Add. Resend will give you TXT and MX records.
3
Add those records in Squarespace DNS Settings (same place as Step 4) under Custom Records.
4
Back in Resend, click Verify — usually confirms within a few minutes.
5
Contact your developer — they'll update one line of code to switch the contact form from test mode to live, sending directly to support@midwestern.construction.

How to Update Your Website Content

Update Blog Post Images

1
Go to midwestern-roofing.sanity.studio and log in.
2
Click Blog Post → select the post → upload a new image → click Publish.
3
To push the update live, call the bake endpoint by visiting this URL in your browser or sending a POST request:
POST https://midwestern.construction/api/bake
Your developer can set this up as a Sanity webhook so it happens automatically.

Update Contact Info, Phone, Address

Contact your developer — these can be updated in Sanity or directly in the code depending on the field.

  • DNS records added in Squarespace (A record + CNAME)
  • midwestern.construction domain verified as live in Vercel

  • Google Ads & Analytics IDs provided to developer
  • Test contact form submission received at support@midwestern.construction
